多表查询：
	笛卡尔积 ： 有两个集合A B取这两个结合在一起的所有集合（A集合*B集合  一般不这么做 要加where）
			我们一般是查交集部分
	多表查询的分类
		内连接
			隐式内连接：使用wherever消除无用数据
			显示内连接：语法：select 字段 from 表名 inner join 表名 on 条件 where 条件
			内连接 注意事项：从哪里查询出数据， 条件是什么  差哪些字段 
					这些取得的数据是交集部分 就是两个集合都有的数据
		外链接
			左外连接
				语法：select 字段 from 左表表名 left [outer] join 右表表名 on 条件  where 条件 
					左表为主表  查询的是左表的所有数据以及他们的交集部分
			右外连接
				语法：select 字段 from 左表表名 right [outer] join 右表表名 on 条件  where 条件 
					右表为主表   查询的是右表的所有数据以及他们的交集部分
		子查询
			概念：查询中嵌套查询，称嵌套查询为子查询
		子查询的不同情况
			子查询的结果是单行单列的
				子查询可以作为条件 ，使用运算符去判断 。关系运算符  > >= < <= != =
			子查询结果是多行单列
				子查询可以作为条件 ，使用IN来判断
			子查询结果是多行多列
				子查询可以作为一张虚拟表，参与查询          
		（#################）多表左外连接 select 字段 from 表名 left join 表名 on 条件  left join 表名 on 条件
		
事物：
	事物基本介绍：
		概念：如果一个包含多个步骤的业务操作，被事物管理，那么这些操作要目同时成功，要么同时失败
		操作：
			开启事物 start transaction
			回滚	rollback
			提交  	commit
				事物 
					有手动提交 手动提交需要写 start transaction 开启事物
					自动提交  mysql是自动提交（每执行一条sql语句 就会自动提交一次） Oracle 默认是手动提交 （必须要写commit）
						修改默认 set @@autocommit=0 0代表手动提交 1代表自动提交 查看是 select @@autocommit；
	事物的四大特征：（重要）
		原子性：事物是原子性的，是不可分割的最小单位，要么同时成功，要么同时失败，（只能把它看做一个整体，不可再分割）
		持久性：当事物提交或回滚后，数据库会持久化的保存数据
		隔离性：多个事物之间，相互独立。
		一致性：事物操作前后，数据总量不变。（数据总量不变）
			
	事物的隔离级别：（了解）
		概念：多个事物之间是隔离的，相互独立的。但是如果多个事物操作同一批数据，则引发一些问题，设置不同的隔离级别就可以解决这些问题
			脏读：一个事物，读取到另一个事物中没有提交的数据
			不可重复读：在同一个事物中，两次读取到的数据不一样
			幻读：一个事物操作（DML）数据表中所有记录，另一个事物添加量一条数据，则第一个事物查询不到自己的修改

		隔离级别
			read uncommitted：读未提交  问题 脏读，不可重复读 ，幻读
			readcommitted：读已提交	    问题 不可重复读  ，幻读
			repeatable read 可重复读    问题  幻读
			serializable 串行化	 没有问题	
		注意 隔离级别从小到大安全性越来越高，但效率越来越低
			查询数据库隔离级别 select @@tx_isolation； 		修改set global transaction isolation level read 隔离级别；
